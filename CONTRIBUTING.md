# CONTRIBUTING

Thank you for contributing to DEVO-AI-OS.

## Contribution Types

- New engine
- New workflow
- New template
- New world bible layer
- Prompt archive example
- Quality checklist improvement
- Tool-specific adaptation

## New Engine Rule

Every new engine must include:

1. Purpose
2. Input
3. Output
4. Tools
5. Workflow
6. Template
7. Quality Checklist
8. Examples

## Naming Convention

Use uppercase engine names:

```text
BOOK_ENGINE.md
DOCUMENTARY_ENGINE.md
PODCAST_ENGINE.md
COURSE_ENGINE.md
```

## Writing Style

- Clear
- Practical
- Visual-first
- Tool-agnostic
- Beginner-friendly
- Expandable

## Pull Request Checklist

- [ ] Does this fit the core philosophy?
- [ ] Is the structure reusable?
- [ ] Is it tool-agnostic where possible?
- [ ] Is there a quality checklist?
- [ ] Can a beginner understand it?
